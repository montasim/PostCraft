# LinkedIQ Frontend Clone Specification

**Purpose:** Any AI agent can use this spec + `DESIGN_SYSTEM.md` + `requirements.md` to build a pixel-perfect clone of the LinkedIQ dashboard using Next.js App Router, shadcn/ui, and TypeScript.

**Source framework:** TanStack Start + Vite → **Target framework:** Next.js 15 App Router

---

## 1. Project Setup

```bash
# Create Next.js app
npx create-next-app@latest linkediq --typescript --tailwind --eslint --app --src-dir

# Install shadcn/ui
npx shadcn@latest init

# Install required shadcn components
npx shadcn@latest add button card badge avatar textarea tooltip separator breadcrumb switch \
  dropdown-menu sheet popover command input label skeleton progress tabs

# Install additional dependencies
npm install lucide-react sonner cmdk class-variance-authority clsx tailwind-merge zod
```

## 6. Custom Components

### 6a. MultiSelect (`src/components/shared/multi-select.tsx`)

```tsx
"use client";

import * as React from "react";
import { Check, ChevronsUpDown, X } from "lucide-react";
import { cn } from "@/lib/utils";
import { Badge } from "@/components/ui/badge";
import {
  Command,
  CommandGroup,
  CommandInput,
  CommandItem,
  CommandList,
} from "@/components/ui/command";
import {
  Popover,
  PopoverContent,
  PopoverTrigger,
} from "@/components/ui/popover";

interface MultiSelectProps {
  options: string[];
  selected: string[];
  onChange: (selected: string[]) => void;
  placeholder?: string;
  className?: string;
}

function MultiSelect({
  options,
  selected,
  onChange,
  placeholder = "Select...",
  className,
}: MultiSelectProps) {
  const [open, setOpen] = React.useState(false);

  const toggle = (value: string) => {
    onChange(
      selected.includes(value)
        ? selected.filter((s) => s !== value)
        : [...selected, value],
    );
  };

  return (
    <Popover open={open} onOpenChange={setOpen}>
      <PopoverTrigger asChild>
        <div
          role="button"
          tabIndex={0}
          className={cn(
            "flex min-h-[44px] max-h-[88px] cursor-pointer flex-wrap items-center gap-1.5 overflow-y-auto overflow-x-hidden rounded-lg border border-input bg-background p-2 text-sm",
            className,
          )}
        >
          {selected.length > 0 ? (
            selected.map((s) => (
              <Badge
                key={s}
                variant="secondary"
                className="gap-1 rounded-md bg-primary/20 text-primary"
              >
                {s}
                <button
                  className="ml-0.5 rounded-full outline-none ring-offset-background focus:ring-2 focus:ring-ring focus:ring-offset-2"
                  onKeyDown={(e) => {
                    if (e.key === "Enter") {
                      toggle(s);
                    }
                  }}
                  onMouseDown={(e) => {
                    e.preventDefault();
                    e.stopPropagation();
                  }}
                  onClick={(e) => {
                    e.preventDefault();
                    e.stopPropagation();
                    toggle(s);
                  }}
                >
                  <X className="h-3 w-3" />
                </button>
              </Badge>
            ))
          ) : (
            <span className="text-muted-foreground">{placeholder}</span>
          )}
          <ChevronsUpDown className="ml-auto h-4 w-4 shrink-0 opacity-50" />
        </div>
      </PopoverTrigger>
      <PopoverContent className="w-[--radix-popover-trigger-width] p-0" align="start">
        <Command>
          <CommandInput placeholder="Search..." />
          <CommandList>
            {options.map((option) => (
              <CommandItem
                key={option}
                value={option}
                onSelect={() => toggle(option)}
              >
                <Check
                  className={cn(
                    "mr-2 h-4 w-4",
                    selected.includes(option) ? "opacity-100" : "opacity-0",
                  )}
                />
                {option}
              </CommandItem>
            ))}
          </CommandList>
        </Command>
      </PopoverContent>
    </Popover>
  );
}

export { MultiSelect };
```

```tsx
"use client";

import { cn } from "@/lib/utils";
import type { NavItem } from "@/types";

interface NavGroupProps {
  label: string;
  items: NavItem[];
  active: string;
  onSelect: (id: string) => void;
}

function NavGroup({ label, items, active, onSelect }: NavGroupProps) {
  return (
    <div>
      <p className="mb-2 px-3 text-xs font-semibold uppercase tracking-wider text-muted-foreground">
        {label}
      </p>
      <ul className="space-y-1">
        {items.map((it) => {
          const Icon = it.icon;
          const isActive = active === it.id;
          return (
            <li key={it.id}>
              <button
                onClick={() => onSelect(it.id)}
                className={cn(
                  "flex w-full items-center gap-3 rounded-lg px-3 py-2 text-sm transition",
                  isActive
                    ? "bg-sidebar-accent text-primary font-medium"
                    : "text-sidebar-foreground hover:bg-sidebar-accent/50",
                )}
              >
                <Icon className="h-4 w-4" />
                {it.label}
              </button>
            </li>
          );
        })}
      </ul>
    </div>
  );
}

export { NavGroup };
```

## 11. Theme Switching

- Default theme: dark (class `:root`)
- Toggle switches between `.light` and removes it (falls back to `:root`)
- Theme toggle button in header: `<Sun>` for dark mode, `<Moon>` for light mode
- Implementation: `document.documentElement.classList.toggle("light", theme === "light")`
- All components use semantic tokens (`bg-background`, `text-foreground`) — theme-aware automatically

## 14. Engineering Principles (MANDATORY)

The agent MUST follow these principles. Reference `DESIGN_SYSTEM.md` sections 15–19 for full details.

### SOLID in React

| Principle | Rule | Example |
|-----------|------|---------|
| **S** — Single Responsibility | One concern per component. Extract when > 150 lines or doing 2+ things | `PlanQuotaCard` only handles quota display. `BrandCard` only shows brand. |
| **O** — Open/Closed | Extend via composition + children. Never modify shadcn primitives | Compose `<Card>` with children, don't fork `card.tsx` |
| **L** — Liskov | Shared interfaces for similar components | All ranked items implement `RankedItemProps` |
| **I** — Interface Segregation | Small, focused props. `Pick<>` over full objects | `<ScorePill label="Score" short="S" value={91} color="chart-5" />` not `<ScorePill variant={fullVariantObject} />` |
| **D** — Dependency Inversion | Data via props/hooks. Never import services in components | `copyPost` handler passed as `onCopy` prop to `VariantCard` |

### Clean Code Rules

| Rule | Enforcement |
|------|-------------|
| Naming | PascalCase components, camelCase functions, UPPER_SNAKE constants |
| No magic numbers | Extract to `src/lib/constants.ts` |
| Max 150 lines | Split into sub-components when exceeded |
| Early returns | Guard clauses first, no deep nesting |
| No `any` types | Use Zod for runtime validation |
| Class merging | Always use `cn()`, never string concat |
| Barrel exports | Every module exports via `index.ts` |

### Next.js Best Practices

| Practice | Rule |
|----------|------|
| Server Components default | No `"use client"` unless state/effects/browser APIs needed |
| Client boundary | Only leaf interactive components get `"use client"` — never layout/page |
| Data fetching | Server components fetch directly, client uses React Query |
| Error boundary | `error.tsx` at route level |
| Loading state | `loading.tsx` with Skeleton |
| Colocation | Tests + types next to components |

## 15. Custom Hooks (Extracted State Logic)

### 15a. `src/hooks/use-theme.ts`

```typescript
"use client";

import { useState, useEffect, useCallback } from "react";

type Theme = "dark" | "light";

function useTheme(defaultTheme: Theme = "dark") {
  const [theme, setTheme] = useState<Theme>(defaultTheme);

  useEffect(() => {
    const root = document.documentElement;
    root.classList.toggle("light", theme === "light");
  }, [theme]);

  const toggle = useCallback(() => {
    setTheme((prev) => (prev === "dark" ? "light" : "dark"));
  }, []);

  return { theme, setTheme, toggle };
}

export { useTheme };
export type { Theme };
```

### 15b. `src/hooks/use-clipboard.ts`

```typescript
"use client";

import { useState, useCallback } from "react";
import { toast } from "sonner";

const RESET_DELAY_MS = 1500;

function useClipboard() {
  const [copiedId, setCopiedId] = useState<number | null>(null);

  const copy = useCallback((text: string, id: number) => {
    navigator.clipboard.writeText(text);
    setCopiedId(id);
    toast.success("Copied to clipboard!");
    setTimeout(() => setCopiedId(null), RESET_DELAY_MS);
  }, []);

  const isCopied = useCallback((id: number) => copiedId === id, [copiedId]);

  return { copy, isCopied };
}

export { useClipboard };
```

### 15c. `src/hooks/use-carousel.ts`

```typescript
"use client";

import { useRef, useCallback } from "react";

function useCarousel() {
  const ref = useRef<HTMLDivElement>(null);

  const scroll = useCallback((direction: "left" | "right") => {
    if (!ref.current) return;
    const delta = ref.current.offsetWidth * 0.5;
    ref.current.scrollBy({
      left: direction === "left" ? -delta : delta,
      behavior: "smooth",
    });
  }, []);

  return { ref, scrollLeft: () => scroll("left"), scrollRight: () => scroll("right") };
}

export { useCarousel };
```

### 15d. `src/providers/theme-provider.tsx`

```tsx
"use client";

import { createContext, useContext } from "react";
import { useTheme, type Theme } from "@/hooks/use-theme";

interface ThemeContextValue {
  theme: Theme;
  toggle: () => void;
}

const ThemeContext = createContext<ThemeContextValue | null>(null);

function ThemeProvider({ children }: { children: React.ReactNode }) {
  const { theme, toggle } = useTheme();
  return (
    <ThemeContext.Provider value={{ theme, toggle }}>
      {children}
    </ThemeContext.Provider>
  );
}

function useThemeContext() {
  const ctx = useContext(ThemeContext);
  if (!ctx) throw new Error("useThemeContext must be used within ThemeProvider");
  return ctx;
}

export { ThemeProvider, useThemeContext };
```

```typescript
export function getQuotaMessage(used: number, limit: number): string {
  if (used === 0) return "Your ideas deserve great posts — start now";
  if (used < Math.floor(limit / 2)) return "Good progress — keep the creative flow going";
  if (used < limit) return "Running low — upgrade for unlimited";
  return "Upgrade your plan to unlock more";
}

export function getQuotaFooter(used: number, limit: number): string {
  if (used === limit) return "Upgrade for unlimited generations";
  if (used >= limit - 1) return "1 generation remaining";
  return `${used} of ${limit} used this period`;
}
```

## 18. State Management Strategy

### Decision Matrix

| State | Owner | Location |
|-------|-------|----------|
| Theme (dark/light) | Global context | `ThemeProvider` in `providers/` |
| Active nav item | Local | `AppShell` (shared by sidebar + header) |
| Mobile sidebar open | Local | `AppShell` |
| Topic text | Local | `TrendInputCard` |
| Audience/Tones/Languages | Local | `TrendInputCard` |
| Emoji toggle | Local | `TrendInputCard` |
| Copied state | Hook | `useClipboard` hook |
| Carousel ref | Hook | `useCarousel` hook |

### Rules

1. **Start local** — only lift state when 2+ siblings need it
2. **Never duplicate derived state** — compute on the fly
3. **URL state for shareable filters** — `useSearchParams()` for page/sort
4. **Context only for global** — theme is the only truly global state
5. **Server state via React Query** — when API connected, never useState for fetched data

### API Integration Points

This spec uses mock data for standalone frontend development. When connecting to the backend (per `REQUIREMENTS.md`), replace mock data at these integration points:

| Current Mock | Backend Integration | Location |
|-------------|--------------------|----------|
| `VARIANTS` constant | `POST /api/variants` via React Query | `src/lib/constants.ts` → `src/hooks/use-posts.ts` |
| `POSTS_USED` constant | `GET /api/workspace/usage` via React Query | `src/lib/constants.ts` → `src/hooks/use-usage.ts` |
| Static guardrails | `GET /api/guardrails?workspaceId=X` | `src/components/features/generate/guardrails-panel.tsx` |
| Topic form submit | `POST /api/trends` + queue polling | `src/components/features/generate/trend-input-card.tsx` |

All API calls should use the service layer pattern from `REQUIREMENTS.md` section 10 (Controller → Service → Repository). Client components should never call APIs directly; use custom hooks that abstract React Query calls.
